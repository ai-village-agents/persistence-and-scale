## Most Promising Concepts (Specific + Actionable):

### A. "How Git Actually Works: The Three Trees"
**Learning Objective:** Explain how Git's three-tree architecture (working directory, staging area, repository) enables version control

**Audience:** Developers who use Git commands but don't understand the underlying model

**Why it matters:** Understanding the model makes Git commands make sense instead of feeling like magic

**Visual approach:** Animated diagrams showing how changes flow through the three trees

**Length:** 4-5 minutes

### B. "Why Writing is Thinking: The Blank Page Test"
**Learning Objective:** Demonstrate how writing reveals gaps in understanding through practical example

**Audience:** Students, knowledge workers, anyone learning complex topics

**Why it matters:** Many people read passively and think they understand, but writing forces clarity

**Visual approach:** Side-by-side: reading a passage vs writing about it, showing where understanding breaks down

**Length:** 3-4 minutes

### C. "Creative Constraints: Why the Haiku Works"
**Learning Objective:** Explain how constraints (5-7-5 syllables) force creative choices using haiku as case study

**Audience:** Anyone interested in creativity, writing, art, problem-solving

**Why it matters:** Shows constraints aren't limitations - they're design tools

**Visual approach:** Show multiple haiku, highlight choices made within constraint, contrast with unconstrained free verse

**Length:** 3-4 minutes

### D. "Understanding Unix Pipes: Data as Water"
**Learning Objective:** Explain Unix pipe philosophy using water/plumbing analogy to make command composition intuitive

**Audience:** New programmers, developers transitioning to Unix systems

**Why it matters:** Pipes are powerful but abstract - good mental model makes them click

**Visual approach:** Animated water flowing through pipes, gradually replacing with data

**Length:** 4-5 minutes
